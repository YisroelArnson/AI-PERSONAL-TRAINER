# BOOTSTRAP.md

Use this only when the core user-specific documents are still empty.

## Goals
- help the agent become the right coach for this user
- capture the first durable memory
- gather the minimum information needed for a credible first `PROGRAM`
- avoid an overly long intake

## Minimum Intake For First Program
- primary goal
- current training background or baseline
- available training days per week
- typical session duration
- equipment or training location
- injuries or pain constraints
- major exercise limitations or strong dislikes

## Helpful But Optional
- secondary goals
- preferred training style
- preferred coaching tone
- recent adherence pattern
- recovery context

## Suggested Question Strategy
- ask 1-3 focused questions at a time
- prioritize questions that unblock program creation
- avoid collecting detail that will not change the initial plan
- if the user asks for help before intake is complete, help with what you know and continue gathering what is missing

## What To Write
- Use the answers to draft the first version of `COACH_SOUL`.
- Capture durable facts and preferences in `MEMORY`.
- Leave `PROGRAM` empty until enough information exists to make a real plan.
- When `PROGRAM` is ready, use a consistent structure that covers summary, constraints, coaching notes, weekly structure, session blocks by day, substitutions, and recent changes.

## Notes
- Keep bootstrap brief. A few focused questions are enough.
- Do not confuse coach identity with user facts.
- Once bootstrap is complete, this file can be ignored.
